Tower Game DESIGN SPEC

Date: 2026-03-29 | Status: Ready for Implementation Planning

Overview

Tower Game is a second multiplayer mini-game in the bar app, living alongside Drink Roulette on the table menu page. Players take turns holding a button to fill a vertical tower meter. The fill accelerates exponentially — deceptively slow at the start, terrifyingly fast at the end. Get as close to the 82% target line as possible without overflowing. The closest safe fill wins the round and assigns a social forfeit to anyone at the table.

Integration: Same table session model as Drink Roulette. Same lobby flow, same Pusher channel, new Redis key table:{tableId}:tower. Cannot run simultaneously with Drink Roulette.

The Fill Curve

The most important design decision. The fill rate must feel exponential — manageable for the first 3 seconds, then suddenly uncontrollable.

fill(t) = -1/k · ln(1 - k · v · t)

where: v = 0.08 (base speed), k = 3 (acceleration), bust at t = 1/(k·v) ≈ 4.17s

The instantaneous fill rate is v · e^(k · fill) — meaning the tower accelerates based on how full it already is, not just time elapsed. A nearly-full tower fills itself.

Hold TimeFill %Player Experience
1.0s9%Slow, safe — builds confidence
2.0s22%Still easy
3.0s42%Halfway point — still feelable
3.5s61%Acceleration becoming noticeable
3.8s81%Skill zone — right at target
4.0s105%Bust — happened in 0.2 seconds

The window between "hit the target" and "bust" is approximately 0.2 seconds. This is the chaos that makes the game fun to watch.

Rules

  • Target line is fixed at 82% every round.
  • Players take turns in randomized order — one turn per player per round.
  • On your turn: hold the button to fill, release to lock in your fill amount.
  • Safe: fill ≤ 100% → score = fill %. Closest to 82% wins.
  • Bust: fill > 100% → score = 0 pts.
  • Round winner = highest score without busting.
  • Winner assigns the social forfeit to any other player.

State Machine

LOBBY → ROUND_STARTING → PLAYER_TURN → TURN_RESULT → (repeat) → ROUND_END → FORFEIT → LOBBY

Player Flow

  1. Host taps "Play Tower Game 🗼" on the table menu page.
  2. Tower Game lobby overlay appears. Other players at the same table see it and join by entering a nickname/emoji (same flow as Drink Roulette).
  3. Host clicks "Start Round" (requires ≥2 players). Turn order randomized server-side.
  4. Each player's turn: 3-second countdown → full-screen tower → hold button → release → result shown 2s → next player.
  5. After all players: leaderboard screen. Winner announced.
  6. Winner taps a player to assign forfeit. Forfeit badge shown on that player's screen.
  7. Play Again resets to Lobby with same players.

Scoring

  • Per turn: score = fill% (0–100) if not busted; score = 0 if busted.
  • Round winner = highest score.
  • Tiebreaker = player who took their turn later in the round wins (rewards risk-taking under pressure).
  • Optional session score across rounds: win = +2 pts; fill within ±2% of target = +1 bonus pt.

Architecture

  • Fill physics: 100% client-side. Server never touches timing during a hold.
  • Redis key: table:{tableId}:tower (1hr TTL, same pattern as :game)
  • Pusher channel: table-{tableId} (shared with Drink Roulette)
  • New Pusher events: tower-updated, tower-turn-start, tower-turn-result, tower-round-end, tower-forfeit
  • New API routes: /api/tower/create, join, start-round, submit-turn, assign-forfeit, [tableId]
  • Server validates: busted = (fill >= 1.0) on submit to prevent spoofing

UI/UX Design System

  • Style: Matches existing Cyberpunk / OLED Dark Mode
  • Tower fill gradient: Violet (bottom) → Indigo → Amber → Rose (top) — color reflects danger visually
  • Danger zone (80–100%): Red overlay + pulsing glow; tower shakes at ≥75% fill
  • Target line: Emerald (#10B981) with glow, fixed at 82% height
  • Bust screen: Rose full-screen background (same as Drink Roulette loser screen)
  • Win screen: Violet gradient (winner picks forfeit, not emerald — keeps consistent with primary accent)

✓ Architecture fits cleanly into existing Redis/Pusher/Next.js stack. No new dependencies required. Fill curve validated for ~4.2s bust window with meaningful 0.2s skill window around the 82% target.
